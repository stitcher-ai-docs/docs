Short definitions for the terms used throughout these docs. Each term is defined once here. Other pages link their first mention back to this page rather than restating it.

## Platform and structure

<ResponseField name="Environment">
  The top-level workspace that holds your pipelines and data connections.
</ResponseField>

<ResponseField name="Pipeline">
  It defines how data flows from raw input to curated output. It hosts a single cost model, backed by a GitHub repository.
</ResponseField>

<ResponseField name="Repository">
  The GitHub repository behind a pipeline, where the cost model configuration lives and where the versions are controled.
</ResponseField>

<ResponseField name="Data Connection">
  A configured connection to a data source or a destination.
</ResponseField>

<ResponseField name="Authoring environment">
  Where you build the cost model: the AI chat, the visual panel, or YAML file.
</ResponseField>

<ResponseField name="Workflow">
  A scheduled or triggered execution of a pipeline.
</ResponseField>

<ResponseField name="Run">
  A single execution of a pipeline.
</ResponseField>

## The four stages

<ResponseField name="Stage">
  Each one of the four phases of the cost modeling: Connect, Model, Build, and Engage.
</ResponseField>

<ResponseField name="Connect">
  The stage that ingests raw cost and business data into the pipeline.
</ResponseField>

<ResponseField name="Model">
  The stage that shapes the data, through Prepare, Normalize, Enrich, and Allocate.
</ResponseField>

<ResponseField name="Build">
  The stage that produces outputs as a data artifacts, through Generate and Govern.
</ResponseField>

<ResponseField name="Engage">
  The stage that delivers the data artifacts created from the modeled data to where stakeholders consume them.
</ResponseField>

## Modeling concepts

<ResponseField name="Cost data">
  The raw cost and usage data the model is built on, such as cloud billing, AI provider cost and usage, and SaaS.
</ResponseField>

<ResponseField name="Business data">
  Data that adds business context to cost, such as ownership, teams, revenue, and budgets.
</ResponseField>

<ResponseField name="FOCUS">
  The open FinOps Cost and Usage Specification your data is normalized to.
</ResponseField>

<ResponseField name="Prepare">
  Clean and structure raw source data before it is normalized to FOCUS.
</ResponseField>

<ResponseField name="Normalize">
  Convert source data into the FOCUS schema.
</ResponseField>

<ResponseField name="Enrich">
  Add business context to normalized data.
</ResponseField>

<ResponseField name="Allocate">
  Assign shared and untagged costs to owners.
</ResponseField>

<ResponseField name="Artifact">
  A generated output of the cost model, such as a chargeback report, a KPI, or a forecast.
</ResponseField>

<ResponseField name="Artifact type">
  The kind of artifact produced, such as a primary dataset, derived dataset, time series indicator, control, or alert.
</ResponseField>

<ResponseField name="Control">
  A rule that governs cost behavior, either proactive or reactive.
</ResponseField>

## Authoring methods

<ResponseField name="AI authoring">
  Build the cost model through the AI chat in natural language.
</ResponseField>

<ResponseField name="Visual authoring">
  Configure stages through the left panel, with an assisted UI.
</ResponseField>

<ResponseField name="YAML authoring">
  Edit the pipeline configuration directly in YAML.
</ResponseField>

## Concepts

<ResponseField name="Finance as Code">

</ResponseField>
