## Generate — produce artifacts

**Generate** transforms and runs calculations on the primary cost dataset and on other artifacts to produce new, specialized **artifacts**. It also **aggregates** detailed cost — sum, average, percentiles, distinct count, and more — to roll it up into those artifacts.

The artifact types you can generate:

- **Indicators / KPIs** — a metric tracked over time at a chosen granularity and grouping, optionally linked to a budget for budget-vs-actual variance.
- **Derived datasets** — a custom view built with a query over the cost model and other artifacts (chargeback tables, margin by feature, cost per tenant).
- **Commitment-coverage reports** — reserved-instance and savings-plan coverage and utilization.
- **Forecasts** — projections built on an indicator (see Event, below, and [Forecasting](/forecasting)).

Artifacts can be **chained** — built from the primary cost dataset or from other artifacts — and each carries a **group** and a **name** you assign, which you use to select it downstream.

## Govern — evaluate against controls

**Govern** evaluates the artifacts produced earlier against configurable **controls** and produces **control artifacts** that report pass or fail. You select the artifact to govern by its **group and name** — the same selection model used when chaining artifacts — and a breach raises an **alert** routed to the right owner.

## Event — define forecast scenarios

**Event** defines named **event banks** — collections of described business changes and what-ifs (a launch, a migration, a contract change, a past anomaly) that **feed forecasts**. Toggling an event bank on or off lets you compare what-if scenarios side by side.
