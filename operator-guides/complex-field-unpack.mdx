A **Complex Field Unpack** operation explodes keys from an **object / map** column — for example a provider `Tags` or `SkuPriceDetails` field — into their own columns. Each key you select becomes a separate column you can then filter, map, or attribute on.

For a **JSON value stored as text**, use **JSON Field Unpack** instead — same mechanics, different source type.

**Availability.** Runs in both **Prepare** and **Enrich**. In Prepare it flattens raw provider blobs before FOCUS exists; in Enrich it works on FOCUS-era data.

<Note>
  Choosing the operation (Step 1) and defining [scope](/pipeline-overview#building-blocks-shared-across-stages) (Step 2) are identical across every operator. This guide covers only **Step 3: Configure field unpack**. Unlike column-writing operators, field unpack has **no merge policy** — each selected key is written to its own target column.
</Note>

## Configure the unpack

<Steps>
  <Step title="Pick the source column">
    Set **Source column name** to the complex column to unpack — a Map, struct, or JSON field such as `Tags` or `SkuPriceDetails`.
  </Step>
  <Step title="Add a key selection rule">
    Under **Key selection rule**, add one or more rules. Each rule has a **Selection type**:

    - **Single key** — extract exactly one key by its exact, case-sensitive name, into one named column.
    - **List of keys** — extract several keys by exact name; each becomes its own column.
    - **Key pattern** — extract every key whose name matches a regular expression; each becomes its own column, named from the key (with optional prefix/suffix).
  </Step>
  <Step title="Name the target column(s)">
    For each rule, set the **Target configuration**: **Column type** (FOCUS or Custom — custom columns start with `x_`) and **Target column name**. Optionally enable a **target column transformation** to derive the value with an expression, or a **target column default value** used when the key is absent.
  </Step>
</Steps>

## Worked example — flatten a `team` tag

Extract the `team` key from a provider `Tags` map into its own attribution column:

- **Source column name:** `Tags`
- **Selection type:** Single key · **Key value:** `team`
- **Column type:** Custom · **Target column name:** `x_team`
- **Enable target column default value:** `unassigned` (for rows whose tag is missing)

Every row's `team` tag now lives in its own `x_team` column, ready to attribute on.

## Result & gotchas

- Key names are **exact and case-sensitive** for Single key and List of keys — use **Key pattern** to catch variants by regular expression.
- With **Key pattern**, each matched key becomes its own column; use the prefix/suffix to keep names tidy and avoid collisions.
- There's **no merge policy** on field unpack — pick target column names that don't collide with columns already in the dataset.
