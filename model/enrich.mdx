Once data is in FOCUS, **Enrich** adds business meaning on top — and this is where **attribution** mostly happens: the lookups and mappings that attach a team, product, or owner to each cost row.

## Attribution — assigning cost to an owner

<CardGroup cols={2}>
  <Card title="Attribution" icon="user-tag">
    Assign cost — _including to shared components_ — so every component, shared ones included, has a clear view of its own cost.
  </Card>

  <Card title="Allocate" icon="share-nodes" href="/model/allocate">
    Distribute the _shared portions_ of cost to a set of receivers. Covered in depth on the [Allocate](/model/allocate) page.
  </Card>
</CardGroup>

Most attribution happens here, as cost data is enriched. A **Lookup** joins an ownership table onto each cost row by a key; a **Mapping** derives an owner from rules on the values already present. After enrichment, most cost carries the owner it belongs to.

What's left is the shared, central, and untagged cost that no single owner can claim — a Kubernetes platform, an API gateway, an enterprise license. That is what [Allocate](/model/allocate) distributes. Attribution narrows; allocation spreads what remains.

<Note>
  The hand-off is the point: the rows attribution can't assign to one owner are exactly the rows allocation distributes.
</Note>

## The operator catalog

Each operator below also runs in [Prepare](/model/prepare) unless it's marked Enrich-only. All of them use the [shared building blocks](/pipeline-overview#building-blocks-shared-across-stages) — scope, conditions, and merge behavior.

| Operator | What it does | When it's useful |
| --- | --- | --- |
| **Mapping** | Adds or updates a column from an **ordered set of conditions**, with a default value for rows that match none. | Derive `x_team` or `x_cost_center` from rules. |
| **Lookup** | Merges columns from a **business dataset** into the cost dataset by matching values in a join column; you can rename imported columns and choose how to handle duplicate matches. | Attribute resources to owners via a mapping table. |
| **Unpack object value** | Explodes keys from an **object/map field** into new columns — a single named key, a list of keys, or all keys matching a pattern. | Flatten provider tag or label blobs. |
| **Unpack JSON value** | The same, for a **JSON value stored as text**. | Flatten a JSON metadata field. |
| **Compute column** | Sets a column to a **static value**, a **column reference**, or an **expression** derived at runtime with SQL-style functions. | Derived metrics, constants, calculated fields. |
| **Filter rows** | **Keeps or removes** rows that match one or more conditions. | Drop test namespaces or non-billable rows. |
| **Remove columns** | Drops **non-FOCUS columns**. FOCUS columns are protected and cannot be removed. | Shed sensitive or oversized raw fields. |
| **Simulate periodic cost**_(Enrich only)_ | Projects **recent or historical cost forward** so in-month spend isn't a surprise. Source the projection from the previous month or a historical average. | End-of-month, quarterly, or late-arriving invoices. |
| **AI assisted mapping**_(Enrich only)_ | Applies a **custom-trained fuzzy-mapping model** at run time to collapse messy variants into one canonical value, with a minimum-accuracy guard and a fallback for unknowns. | Normalize inconsistent vendor, region, or service names. |

<Info>
  **Amortize cost** — spreading a charge over time, for example recognizing an annual SaaS contract monthly — is **forthcoming**. It is not available yet.
</Info>

<Warning>
  **Unmatched rows are not dropped.** Cost rows that find no matching key in a lookup or mapping are treated as **shared or untagged cost** and handed to [**Allocate**](/model/allocate), which distributes them to the teams that consumed them.
</Warning>
