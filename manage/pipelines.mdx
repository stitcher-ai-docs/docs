A [pipeline](/general/glossary) is a single cost model, backed by a GitHub [repository](/general/glossary). Pipelines live inside an [environment](/general/glossary) and are managed from the **Pipelines** page.

### Pipeline actions

Each pipeline row shows its name, status, repository, and latest run. The action icons let you:

- **Invoke** the pipeline (run it now).
- **View or edit in GitHub** (opens the repository).
- **Schedule** the pipeline (see [Running a pipeline](#running-a-pipeline)).
- **Edit** the pipeline.
- **Delete** the pipeline.

### Running a pipeline

There are three ways to run a pipeline:

- **Standard**: runs the full pipeline, ingesting all configured sources for the whole month. It builds every artifact, evaluates every control, and writes to every configured destination.
- **Debug**: runs a limited scope over a date range you select, so you can iterate quickly.
- **Schedule**: set a specific time for a standard run to happen automatically.

### Reviewing runs

A [run](/general/glossary) is a single execution of a pipeline. Expand a pipeline to see its runs, filtered by **All**, **Running**, **Succeeded**, **Failed**, and **Cancelled**. Each run shows its status, whether it ran in debug, what scheduled it, start time, and duration.

Expand a run to see the months covered, the four [stages](/general/glossary) (Connect, Model, Build, Engage) and their status, inputs, and logs. Select **Open run graph** for a detailed view.

### Configuration

The **Configuration** tab visualizes the pipeline configuration and its version history. Each tagged commit marks a period of the model's history, so you can track how the cost model changed over time and who changed it. Click a commit icon to add or remove tags.

### Execution rules

Execution rules adjust how a pipeline runs for specific billing periods, for example skipping extraction, or skipping a provider when it errors, for a chosen month. A rule applies to the months you select and stays active for a defined window.

The **Execution rules** tab lists these rules, filtered by **Active**, **Inactive**, and **Expired**. Select **Create rule** to add one. Each rule shows its provider, name, type, status, invoke window, and billing period.
