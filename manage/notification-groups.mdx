A notification group is a named set of targets that StitcherAI notifies when an artifact is exported. You manage notification groups from the **Notification Groups** page, then reference them from the [Engage](/building/engage) stage to decide when they fire.

<Note>
  A notification group defines _who_ is notified, not _when_. The trigger is configured in the [Engage](/building/engage) stage: when you add an integrate operation, enable notification, choose the notification groups to alert, and set how the file is delivered (as a link or an attachment).
</Note>

## Create a notification group

<Steps>
  <Step title="Open Notification Groups">
    Select the **\+** to create a new group.
  </Step>
  <Step title="Name the group">
    Enter a **Name** and **Description**.
  </Step>
  <Step title="Add targets">
    Add one or more targets. For each target, choose a **Type** and fill in its connection parameter:

    - **Email**: the email address to notify.
    - **OpenOps Webhook**: the webhook URL to call.
    - **Slack Channel**: the Slack channel to post to.
  </Step>
  <Step title="Save">
    Save the group.
  </Step>
</Steps>

## Manage notification groups

Each group in the list shows its name, description, and number of targets. Expand a group to see each target's type and connection parameter. Use the action icons to edit or delete a group.
