StitcherAI turns raw cost and usage data from any source into a cost model you can trust: version-controlled, auditable, and built on open standards like FOCUS. You connect your cost and business data sources, shape the data into a clean and business-aware model, generate the artifacts your stakeholders need, and deliver them to where work happens.

This page explains how the core pieces fit together. For a definition of any term, see the [Glossary](/general/glossary).

## How the pieces fit together

StitcherAI is organized as a small set of objects that nest inside each other.

<Steps>
  <Step title="Environment">
    The top-level workspace. It holds your [pipelines](/general/glossary) and your [data connections](/general/glossary).
  </Step>
  <Step title="Pipeline">
    A single cost model. Each pipeline is backed by a GitHub [repository](/general/glossary), so the model is stored as configuration you can review, version, and audit. This is what [Finance as Code](/general/glossary) means in practice.
  </Step>
  <Step title="Cost model">
    The logic inside a pipeline, expressed across four [stages](/general/glossary). You author it through AI, a visual panel, or YAML.
  </Step>
</Steps>

## The four stages

Every cost model moves through the same four stages, in order. Each stage hands its output to the next.

<CardGroup cols={2}>
  <Card title="Connect" icon="plug" href="/building/connect">
    Ingest raw cost and business data into the pipeline.
  </Card>

  <Card title="Model" icon="right-left" href="/building/model">
    Shape the data: Prepare, Normalize to [FOCUS](/general/glossary), Enrich, and Allocate.
  </Card>

  <Card title="Build" icon="screwdriver-wrench" href="/building/build">
    Produce outputs: Generate [artifacts](/general/glossary) and Govern with [controls](/general/glossary).
  </Card>

  <Card title="Engage" icon="rocket" href="/building/engage">
    Deliver outputs to where stakeholders consume them.
  </Card>
</CardGroup>
